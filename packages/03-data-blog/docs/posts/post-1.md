# 为什么需要构建期数据加载

在传统的单页应用里，页面数据通常是在浏览器里通过接口请求拿到的。这种做法在内容型站点（文档、博客）上有几个明显的痛点：首屏要多等一个网络往返、爬虫不一定能看到完整内容、还需要额外维护一套后端接口。

VitePress 给出的答案是**构建期数据加载**。它的核心思想很简单：既然博客的文章列表、标签聚合这类数据在打包时就已经确定，那就在 `vitepress build` 执行时把它们计算好，直接内联进生成的 JavaScript 里。

<!-- more -->

这样做的收益是实打实的。页面打开时数据已经在手边，没有额外请求；生成的是纯静态资源，可以直接丢到任意 CDN；由于内容在 HTML 里就渲染好了，搜索引擎和分享卡片都能正确抓取。

实现这一切的载体就是 `.data.ts`（或 `.data.js`）文件。你在其中导出一个带 `load()` 方法的加载器，VitePress 会在构建时调用它，并把返回值作为具名导出 `data` 提供给页面消费。后面的文章会带你一步步把它用起来。
