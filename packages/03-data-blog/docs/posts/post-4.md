# 用标签组织你的博客

当文章越来越多，单纯按时间排列的列表就不够用了。给文章打上标签，再做一个「标签 → 文章」的索引页，是博客最常见的组织方式。

这件事同样可以在构建期完成。我们写一个数据加载器，遍历所有文章的 `frontmatter.tags`，把每个标签下的文章收集到一起，最终产出一个 `Record<string, Post[]>` 形态的映射。

<!-- more -->

实现思路很直接：在 `transform` 里建一个对象，对每篇文章的每个标签做一次「push」。为了页面渲染稳定，最好对标签名排序，对每个标签下的文章按日期倒序排序。

本 demo 的 `docs/tags.data.ts` 就是这样一个加载器，而 `docs/tags.md` 负责把它导出的 `data` 渲染成分组列表。你会发现，同一批 Markdown 文件，通过不同的加载器就能产出文章列表、标签索引两种完全不同的视图——这正是构建期数据加载的灵活之处。
